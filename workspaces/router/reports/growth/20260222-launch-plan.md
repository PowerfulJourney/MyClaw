## 0) 假设与定位（避免跑偏）
> **假设产品形态：浏览器插件/桌面插件（可上架插件市场）**，核心能力是把用户现有环境（浏览器标签页/网页应用）“接入”到 Agent 执行流里。

### ICP（首发 2 个）
1. **AI 重度用户/自动化玩家**：会折腾、愿意试新工具（HN/Reddit/IndieHackers/开源社区）。
2. **效率型知识工作者**：运营/销售/招聘/客服/研究员，需要在网页上重复操作（内容平台、CRM、招聘网站、表单、后台）。

### 一句话价值主张
- **让 Agent 在你已打开的网页里工作**：不换工具，不重登账号，直接在真实页面执行。

## 1) 插件市场上架文案（A/B）
> 用于 Chrome Web Store / VS Code Marketplace / 浏览器/插件市场通用；字段按各市场适配。

### A 版：效率与“接入真实网页”
**标题（≤45字）**：OpenClaw Browser Relay：让 AI 在真实网页替你操作

**一句话短描述（≤132字）**：
一键把当前标签页接入 Agent，让 AI 在你正在用的网站里执行搜索、填写、整理、点击等任务；无需复制粘贴、无需重新登录。

**长描述（要点 bullets）**：
- 连接：一键授权当前标签页（你的账号仍在本地浏览器中）
- 执行：Agent 可读取页面结构并执行操作（点击/输入/抓取）
- 适用：表单填写、信息收集、后台操作、重复流程自动化
- 安全：仅在你授权的标签页生效；可随时断开

**关键词/Tag**：AI agent, automation, productivity, browser assistant, workflow

**首屏 3 个使用场景（配截图/动图）**：
1) “从 10 个网页提取信息→生成表格”
2) “在后台批量更新字段/状态”
3) “把网页内容汇总成报告/邮件草稿”

### B 版：开发者与“可编排”
**标题**：Browser Relay for Agents：把现有网页变成可编排的工具

**短描述**：
把任意网页/内部系统接入 Agent 执行流：在现有登录态下自动点击、填表、抓取并回传结构化数据。

**长描述（要点）**：
- 对开发者：把网页动作变成可复用步骤（适合内部工具自动化）
- 对团队：把 SOP 变成可执行 runbook（降低重复劳动）
- 支持：示例工作流 + 快速开始文档

**A/B 测试指标**：
- 商店页转化率（访问→安装）
- 安装后激活率（首次连接成功）
- “3 个示例任务”完成率

## 2) 首批分发渠道清单（≥10 个）
> 目标：在 14 天内用“可控、可复用”的渠道拿到前 1000 installs。

11. **GitHub README 引流**：开源/文档仓库（哪怕只开 docs+examples），做“安装→快速开始”。

## 4) 首批 1000 安装获取路径（分解到可执行动作）

### 4.1 漏斗设定（用于估算）
- 商店页访问 → 安装：目标 8–12%
- 安装 → 首次连接成功：目标 45–60%
- 连接成功 → 完成示例任务：目标 35–45%

> 估算：要拿 1000 installs，按 10% 转化需约 10,000 商店页访问。

### 4.2 获取路径（四条腿走路）

#### Path A：插件市场自然流量（目标 300 installs）
- 动作：关键词覆盖 + 截图/GIF 优化 + 评价冷启动（见 4.3）
- 里程碑：上架后 48h 内迭代 2 次文案与截图

#### Path B：社区/榜单爆点（目标 400 installs）
- Product Hunt（目标 150）
- Show HN（目标 100）
- Reddit + IH + V2EX（目标 150）
- 动作：每条渠道都用“不同 ICP 的标题与场景”，避免一稿多投

#### Path C：内容连载（目标 200 installs）
- 14 天×每天 1 条短内容，带单一 CTA
- 2 篇长文做“可收藏”的教程/模板

#### Path D：共创/合作（目标 100 installs）
- 找 10 位小 KOL（每位带来 10 installs 即可）
- 或 2 个 newsletter（每次 50 installs）

### 4.3 评价/口碑冷启动（关键）
- 目标：首周拿到 20 条真实评价（含场景），平均 ≥4.6 星
- 方法：
  1) 插件内触发：完成“连接+示例任务”后弹一次评分请求
  2) 社区招募：前 50 名提供“工作流模板/优先支持”
  3) 反向私聊：在评论区/社区回复中引导评价
