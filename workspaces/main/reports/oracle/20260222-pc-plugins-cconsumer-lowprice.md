# PC端优先 / C端 / 插件市场切入 / 先低价：机会点调研（P0）

## 1) 结论（直接指向钱）
1. **Top1 推荐：轻量“浏览器宏录制器 + 本地运行”扩展**（面向个人高频重复网页操作）。用 **$3–5/月** 的“低价可冲动付费”模型切入，差异化在 **本地/隐私/模板化**，避开大厂自动化平台的复杂与高价。
2. 次优机会主要集中在“信息过载（tab/阅读/视频）→输出到个人知识库”链路，但赛道更拥挤；如果做，必须做 **极窄场景 + 极快价值交付**，否则被 Readwise / Eightify / Workona 等吞没。
3. 插件市场优先级：**Chrome Web Store / Edge Add-ons**（装机量+付费转化最直接）；Obsidian/VS Code 也可做分发，但更偏开发者/知识工作者且付费转化更看口碑与功能深度。

## 3) 机会点列表（3–5个，含优先级排序）

### P1（Top1）轻量浏览器宏录制器（Local-first）
- **目标用户**：
  - 个人电商/二手卖家（重复上架、复制商品信息）、求职者（批量投递/填表）、内容搬运/运营（重复发布/后台操作）、学生（重复收集资料）。
- **刚需痛点**：
  - 重复网页操作耗时、易错；大多数自动化工具太重（要云端、要集成、要学习曲线），且隐私担忧（登录信息/表单内容）。
- **付费路径（低价优先）**：
  - Freemium：免费 3 个宏 + 基础录制/回放；
  - Pro：**$3–5/月** 或 **$29–49/年**（解锁：无限宏、定时、数据导出、云同步可选）；
  - 可加一次性买断（本地版）$19–29 作为“低价锚点”。
- **竞品差异**：
  - 对标 Axiom/Bardeen：不做“跨应用工作流平台”，只做“浏览器内宏 + 模板”；
  - **本地执行**（不上传录制内容），主打隐私；
  - 提供“模板市场”（比如：Amazon/闲鱼/小红书/表单填充）做分发与付费升级。
- **实现风险**：
  - 动态页面/反爬（DOM 变动导致回放失败）；
  - Captcha/登录态处理（需要明确“不绕过安全机制”的合规边界）；
  - 扩展权限敏感（用户对“读取网页内容”警惕，转化会受影响）。
- **MVP 范围**（2周可做）：
  1) 录制：click/type/scroll/wait；
  2) 回放：同域名重放 + 简单重试；
  3) 变量：把输入框标成变量（如姓名/邮箱/手机号），回放时批量填充 CSV；
  4) 导出：执行日志 + 失败截图；
  5) 3 个模板：表单批量填充、网页数据抓取到 CSV、后台固定流程。
- **2周验证方案**：
  - D1–D3：落地页 + 30 秒 GIF + 价格（$3.99/mo）+ waitlist；
  - D4–D7：投放/分发：Chrome Web Store 上架（免费版）+ Reddit/少数派/知乎/B站发“宏模板”；
  - D8–D14：只做 1 个垂直模板（如“批量填表”）做到稳定；
  - 指标：安装→激活（录制一次）>25%；激活→付费>1%；付费 ARPPU>$3。

### P2 极简 Tab/Session 打包器（“把当前浏览器工作区一键封装成包”）
- **目标用户**：重度多任务的人（学生/研究者/知识工作者）。
- **痛点**：tab 爆炸；切换任务时丢上下文；现有产品要么太简单（OneTab），要么太重（Workona）。
- **付费路径**：$2–4/月（解锁：跨设备同步、无限历史、团队分享）。
- **竞品差异**：
  - 只做“session 包”：标题/标签/自动提取页面摘要（本地）+ 一键恢复；
  - 加“截止时间提醒”（防止 session 永久堆积）。
- **实现风险**：
  - 与 Chrome 原生同步、其他扩展冲突；
  - 差异化不强，容易沦为同质化。
- **MVP**：保存/恢复 session、自动去重、标签、导出 Markdown。
- **2周验证**：Chrome Web Store 上架 + 关键词 SEO（tab manager/session manager）+ 收集“愿意付费的功能”投票。

---

### P3 YouTube/网课“时间戳笔记 → Obsidian/Notion 一键导出”
- **目标用户**：自学者、考证党、看大量教程的从业者。
- **痛点**：总结工具要么只给泛泛摘要（不复用），要么太贵；真正需要的是“可回看、可检索、可复习”的结构化笔记。
- **付费路径**：$3–6/月（免费：每周 10 条时间戳；付费：不限量 + 模板导出）。
- **竞品差异**：
  - 不强调“AI 大总结”，强调“边看边记 + 自动结构化”；
  - 输出为 Obsidian/Markdown/Anki 卡片。
- **实现风险**：
  - YouTube 字幕/转录接口变动；
  - AI 成本（如做总结）会侵蚀低价策略。
- **MVP**：时间戳+标题+要点（手动/半自动），导出 Markdown；AI 总结作为可选。
- **2周验证**：做 1 个模板（如“课程笔记模板”）+ 在 Obsidian 社区/学习区投放；看导出次数与复访率。

### P4 双语字幕 + 词汇高亮 + 一键 Anki（面向中文学习者的“轻量 Language Reactor 替代”）
- **目标用户**：英语/日语学习者（尤其是“边看剧边背词”）。
- **痛点**：字幕工具要么功能重、要么对中文用户不友好；真正要的是“查词→例句→入库”。
- **付费路径**：$2–5/月（词库/发音/Anki 同步等）。
- **竞品差异**：
  - 更偏“中文学习路径”（词频、四级/考研/托福/日语 JLPT 词表）；
  - 更轻、更便宜。
- **实现风险**：
  - 竞品强（Language Reactor 先发优势）；
  - 版权/字幕源合规。
- **MVP**：单语/双语字幕、悬停释义、Anki 导出。
- **2周验证**：做“考研词表模式”切入，发 B站/小红书，测转化。

### P5 网页表单“智能填充 + 文本片段管理”（个人版、比企业版更轻更便宜）
- **目标用户**：客服兼职、自由职业者、销售/运营个人。
- **痛点**：重复回复/填资料；现有工具偏企业（贵、复杂）。
- **付费路径**：$2–4/月（跨设备同步、团队库另说）。
- **竞品差异**：
  - 不做 CRM，只做浏览器内“片段 + 快捷键 + 表单字段映射”；
  - 可提供“行业片段包”作为增值。
- **实现风险**：
  - 同质化非常严重（Text expander/输入法/剪贴板管理器）。
- **MVP**：片段库、快捷键、变量、简单表单映射。
- **2周验证**：上架 Chrome Web Store；用“客服话术包/求职套磁包”引流。

## 4) Top1 推荐 & 其余放弃理由
### 为什么 Top1（浏览器宏录制器）最值得先做
- **刚需强**：能立刻省时间；用户愿意为“省 1 小时/周”付 $3–5/月。
- **低价可行**：核心功能本地执行，云成本可控；不像 AI 总结类必须烧 token。
- **插件市场匹配**：Chrome Web Store 搜索意图高；模板内容能形成“长尾获客”。

### 其余放弃/后置理由（若资源只够做1个）
- P2 Tab/Session：差异化难做深，容易沦为“又一个 tab manager”。
- P3 视频笔记：赛道拥挤且 AI 成本影响低价；需非常强的输出链路才能赢。
- P4 字幕学习：竞品成熟、版权/字幕源问题复杂，获客渠道更内容化。
- P5 文本片段：替代品太多（输入法、系统剪贴板、各种 expander），需要强渠道或强 bundle 才能跑出来。
